# Deal Progression Reference

How to assess if a deal is advancing.

## Progression Indicators

### Strong Positive Indicators

| Signal | What It Means |
|--------|---------------|
| Next meeting scheduled | Momentum maintained |
| New stakeholders introduced | Expanding buying committee |
| Internal information shared | Trust building |
| Access granted (systems, docs) | Practical investment |
| Timeline confirmed | Real urgency |
| Budget discussed openly | Serious buying intent |
| "We" language | Ownership forming |
| Questions about implementation | Forward thinking |

### Moderate Positive Indicators

| Signal | What It Means |
|--------|---------------|
| Engaged in meeting | Interest present |
| Questions answered | Cooperation |
| Follow-up promised | Some commitment |
| Interest expressed | Potential |

### Neutral Indicators

| Signal | What It Means |
|--------|---------------|
| Information gathered | Process moving |
| Questions asked | Evaluating |
| No objections | Not blocked |

### Negative Indicators

| Signal | What It Means |
|--------|---------------|
| Vague next steps | Losing momentum |
| New objections | Concerns growing |
| Stakeholder disengagement | Interest waning |
| Timeline pushed | Urgency dropping |
| Competitor mentioned positively | Competition risk |
| "We'll get back to you" | Stalling |
| No decision maker present | Not prioritized |

### Strong Negative Indicators

| Signal | What It Means |
|--------|---------------|
| Meeting cancelled | Deprioritized |
| "Not a priority right now" | Dead or dormant |
| Budget concerns | May not be qualified |
| Key stakeholder skeptical | Blocker emerged |
| Competitor preferred | May lose |

## Stage Progression Assessment

### Lead → Target

**Progression criteria:**
- Interest confirmed
- Potential fit established
- Right stakeholder identified
- Worth further investment

**Evidence needed:**
- Positive response to outreach
- Agreement to first meeting
- Initial qualification passed

### Target → Discovery

**Progression criteria:**
- Discovery meeting held
- Pain confirmed
- Stakeholders engaged
- Budget/timeline plausible

**Evidence needed:**
- Completed first discovery
- Champion identified
- Problem worth solving confirmed

### Discovery → Contracting

**Progression criteria:**
- Use case defined
- Technical requirements clear
- Decision process understood
- Budget confirmed
- Timeline committed
- Proposal requested or welcomed

**Evidence needed:**
- Detailed use case documentation
- Technical validation complete
- All decision makers identified
- Budget range confirmed
- Agreement to receive proposal

### Contracting → Production

**Progression criteria:**
- Proposal accepted
- Terms agreed
- Contract signed
- Implementation begins

**Evidence needed:**
- Written acceptance
- Signed contract
- Implementation kickoff scheduled

## After-Meeting Assessment

### Questions to Answer

1. **Did the deal advance?**
   - Yes (moved closer to close)
   - No (stayed in place)
   - Regressed (moved backward)

2. **What's the evidence?**
   - Specific signals from meeting
   - Actions agreed to
   - Information shared

3. **What stage are we in?**
   - Current stage
   - Ready for next stage?
   - What's blocking progression?

4. **What must happen to advance?**
   - Our actions needed
   - Their actions needed
   - External factors

## Progression Scoring

### Meeting Outcome Score

| Score | Meaning | Indicators |
|-------|---------|------------|
| 5 | Excellent | Multiple strong positives, clear next steps |
| 4 | Good | Several positives, deal advancing |
| 3 | Neutral | No clear progress, no regression |
| 2 | Concerning | Some negatives, momentum slowing |
| 1 | Poor | Multiple negatives, deal at risk |

### Trend Assessment

Compare last 3 meetings:
- Improving: Scores trending up
- Stable: Scores consistent
- Declining: Scores trending down

## Red Flags to Watch

### Immediate Attention

| Red Flag | Response |
|----------|----------|
| Decision maker cancelled | Reach out to reschedule urgently |
| New competitor entered | Accelerate differentiation |
| Champion went cold | Direct conversation needed |
| Budget question emerged | Address immediately |
| Timeline pushed significantly | Re-qualify urgency |

### Monitor Closely

| Red Flag | Response |
|----------|----------|
| New stakeholder with questions | Engage and address |
| Technical concerns raised | Schedule technical discussion |
| Comparison to competitor | Provide differentiation |
| Process questions | Clarify buying process |

## Progression Blockers

### Common Blockers

| Blocker | Resolution |
|---------|------------|
| Decision maker not engaged | Get meeting with DM |
| Technical concerns unaddressed | Schedule technical deep-dive |
| ROI not clear | Build business case |
| Budget not approved | Help build justification |
| Scope not defined | Conduct scoping workshop |
| Competition preferred | Differentiate and prove value |
| Internal politics | Engage executive sponsor |

### Blocker Resolution Template

```
Blocker: [What's blocking]
Impact: [How it affects deal]
Owner: [Who can resolve]
Action: [What we'll do]
Timeline: [When to resolve by]
Escalation: [What if unresolved]
```

Blocker: [If any]

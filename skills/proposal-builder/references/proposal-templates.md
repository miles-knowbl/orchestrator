# Proposal Templates

Three ready-to-use templates scaled by engagement complexity.

---

## Standard Proposal Template

Full-length proposal for engagements over $25K or with multiple stakeholders.

**Sections in order:**

| # | Section | Content | Length |
|---|---------|---------|--------|
| 1 | **Executive Summary** | Client problem, proposed solution, expected outcome with one quantified benefit | 150-250 words |
| 2 | **Understanding of the Challenge** | Current state, pain points, cost of inaction. Reference client's own words from discovery | 1-2 paragraphs |
| 3 | **Proposed Solution** | Approach, methodology, key activities. Break into phases if work spans 4+ weeks | Per phase: objective, activities, deliverables |
| 4 | **Scope Definition** | In-scope items, out-of-scope items, assumptions | See scope-definition-guide.md |
| 5 | **Timeline** | Phase table with duration, start, end dates | Table format |
| 6 | **Investment** | Line items per phase, total, payment terms | See pricing-strategies.md |
| 7 | **Team** | Brief bios of key people; focus on relevant experience | 1-2 sentences each |
| 8 | **Why Us** | 2-3 differentiators with social proof or case studies | Short paragraph |
| 9 | **Next Steps** | 2-3 specific actions with dates | Numbered list |

## Brief Proposal Template

For engagements under $25K or when the client wants a fast turnaround.

```markdown
# [Project Name] Proposal
[Client Name] | [Date]

## Summary
[Client] needs [outcome]. We propose [approach] delivered over
[timeframe] for [investment]. This will [quantified benefit].

## Scope
**We will:**
- [Deliverable 1]
- [Deliverable 2]
- [Deliverable 3]

**Not included:**
- [Exclusion 1]
- [Exclusion 2]

## Investment
[Total amount and payment terms in 2-3 sentences]

## Timeline
[Simple timeline: start date, milestones, end date]

## Next Steps
[Single clear call to action with deadline]
```

## Technical Proposal Template

For technical audiences evaluating approach and architecture.

**Sections in order:**

| # | Section | Content |
|---|---------|---------|
| 1 | **Problem Statement** | Technical description of challenge, constraints, architecture limitations, performance requirements |
| 2 | **Technical Approach** | Architecture overview (with diagram ref), technology stack table (layer / tech / rationale), implementation strategy by phase, integration points (system / method / protocol), security considerations, performance targets table (metric / target / measurement) |
| 3 | **Risks and Mitigations** | Table: risk / probability / impact / mitigation |
| 4 | **Deliverables** | Each with acceptance criteria |
| 5 | **Timeline and Investment** | Reference business proposal or include condensed version |

## Template Selection Guide

| Signal | Template |
|--------|----------|
| Multiple stakeholders, formal process | Standard |
| Small engagement, existing relationship | Brief |
| Technical evaluators in the audience | Technical |
| RFP response | Standard + Technical combined |
| Follow-on work with existing client | Brief |
